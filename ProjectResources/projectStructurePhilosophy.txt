Философия структуры проекта EvieEngine:

1. Scenes
Внутри создаете отдельные папки для каждой сцены, внутри папок вы можете хранить саму сцену, но если на этой сцене используются уникальные модели/текстуры/скрипты храните все так-же в папке с вашей сценой, обязательно разделяя все по отдельным папкам, например:
--------------------
Scenes/
|
|___> Level1.Scene
|
|___> Scripts
|   |
|   |___> Level1SceneLogic.cs
|
|___> Textures
|   |
|   |___> Stone.png
--------------------

2. Models
В этой папке вы храните все 3д модели которые могут использоваться в любой сцене проекта, и все это в четкой структуре: под каждую 3д модель создается отдельная папка с ее названием, и внутри вы храните модели, текстуры и прочее, например:
--------------------
Models/
|
|___> Cat
|   |
|   |___> Cat.fbx
|
|___> Textures
|   |
|   |___> CatBody.png
|
|___> Materials
|   |
|   |___> CatMaterial.material
--------------------

3. Textures и UI 
Тут храните текстуры и UI элементы которые не привязаны к текстурам или чему-то еще конкретному. Философия 1 в 1 как у Models

4. Other
Тут все остальное, так-же старайтесь организовать все по отдельным логическим папкам

5. Resources
Тут все что должно подгружаться через Resources.Load()
